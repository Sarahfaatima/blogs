Embedding External Content with iframes in React.js

by Sarah Fatima

The iframe element has been part of HTML for decades, and despite newer alternatives, it remains one of the simplest ways to embed external content into a web application. In modern frontend development with frameworks like React.js, iframes are still relevant for certain use cases where external pages, widgets, or documents need to be displayed without reimplementation.

What is an iframe?

An iframe (inline frame) is an HTML element that allows embedding another HTML page within the current one. Essentially, it creates a "window" into another website or application.

For example: 

<iframe src="https://example.com" width="600" height="400"></iframe>

This renders the page hosted at example.com inside your own webpage.

Why use iframe with React.js?

While React provides powerful ways to build reusable components and manage UI, there are scenarios where iframes are the most practical solution:

  • Embedding external apps or pages – You can integrate third-party content like blogs, dashboards, or maps.
  • Isolated execution – The iframe runs in a sandboxed environment, so external code doesn't interfere with your React components.
  • Legacy system integration – If you need to show existing static pages or older systems inside your new React project, an iframe is straightforward.
  • Quick prototyping – Instead of rewriting content, you can embed it instantly.

Using iframe in React.js

React supports standard HTML tags in JSX, so using an iframe is almost identical to plain HTML. Here's a clean example: 

import React from "react";

export default function BlogEmbed() {
  return (
    <div style={{ width: "100%", textAlign: "center" }}>
      <iframe
        src="https://example.com/blog"
        title="Embedded Blog"
        width="100%"
        height="600"
        style={{ border: "none", borderRadius: "8px" }}
        loading="lazy"
      />
    </div>
  );
}

Key attributes:

src

URL of the page you want to embed.

title

Accessibility attribute (required for React).

width & height

Dimensions of the iframe.

style

CSS to control appearance (e.g., no border).

loading="lazy"

Improves performance by loading only when visible.

Best Practices and Limitations

  • Always provide a title for accessibility and React warnings.
  • Use lazy loading for performance.
  • Apply CSS carefully to avoid double scrollbars.

Content inside the iframe cannot be directly styled or manipulated by your React app due to browser security policies (CORS & sandboxing).

Not ideal for SEO — search engines generally treat iframe content as separate.

Conclusion

In React.js, iframes remain a practical tool for embedding external content quickly and safely. They are particularly useful when integrating blogs, dashboards, videos, or legacy applications into a modern React project.

While not always the most flexible solution, iframes are easy to implement, require minimal setup, and can bridge the gap between existing resources and new React-based UIs.

If you're building a React app and need to display third-party content, don't hesitate to consider iframes — sometimes the simplest solution is still the best one.
